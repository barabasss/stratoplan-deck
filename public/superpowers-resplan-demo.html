Вайбкодинг · Superpowers

Как начать с Superpowers

На примере фичи для приложения «ресурсный план» — выгрузка плана в Excel: как поставить фреймворк, по какому флоу идёт работа и как агент понимает, что пора на следующий шаг.

Superpowers — фреймворк для кодинга с агентом: набор скиллов, которые ведут работу по чёткому флоу. Скиллы срабатывают сами — агент не прыгает сразу в код.
Шаг 0 · установка

Дай агенту ссылку — он подскажет команду

Установка различается по агентам. Идея одна: показываешь агенту репозиторий Superpowers, он говорит, как поставить под твой харнесс. После установки скиллы активны с первого сообщения.

Claude Code
# официальный маркетплейс плагинов
/plugin install superpowers@claude-plugins-official

# или через маркетплейс obra
/plugin marketplace add obra/superpowers-marketplace
/plugin install superpowers@superpowers-marketplace

Для Codex / Cursor / Gemini / Copilot и т.д. — свои команды установки плагина; смысл тот же.

Механика

Как агент понимает, что пора дальше

Работают три механизма вместе.

01

Триггер по описанию

У каждого скилла есть description: "Use when…". Агент сверяет текущую ситуацию с триггером и сам активирует скилл.

02

Явная эстафета

Каждый скилл в конце называет следующий REQUIRED SUB-SKILL. Цепочка зашита в самих скилах, а не угадывается.

03

Гейты пользователя

Между фазами — твоё подтверждение. На блокере агент останавливается и спрашивает, а не идёт наугад.

Флоу · 5 тактов

Тот же путь — и сразу на нашей фиче

Метка у заголовка — где нужен ты, а где агент сам. В рамке — реальный артефакт фичи «экспорт ресурсного плана в Excel».

1

Обсудить

brainstormingнужен ты

Агент не бросается в код: вопросами проясняет, что именно нужно, и собирает из ответов спеку — показывает её небольшими частями, чтобы её реально можно было прочитать. Ты отвечаешь на вопросы и утверждаешь спеку.

на фиче · экспорт в Excel
агент: выгружаем все три листа (проекты / ресурсы / СВОД) или только сводную?
ты: все три, в формате шаблона.
агент: учитываем роль — РП свои проекты, РМ своё подразделение, полный план для ПБК/ДУП?
ты: да, по роли.
агент: значения или формулы? для переносимости предлагаю посчитанные значения.
ты: значения.
→ спека: эндпоинт /export/resplan.xlsx, openpyxl, 3 листа как в шаблоне, фильтр по роли, значения.
2

Ветка

using-git-worktreesагент сам

Заводит отдельную ветку, ставит зависимости, проверяет, что тесты на старте зелёные. На main без твоего согласия не работает.

на фиче · экспорт в Excel
$ worktree → ветка feat/export-resplan-xlsx
$ pip install -r app/requirements.txt
$ baseline: тестов в проекте нет → чисто; pytest добавим в реализации
3

План

writing-plansнужен ты

Агент пишет план: цель, ограничения и мелкие задачи с точными путями файлов и шагами «тест → код». Ты читаешь план и даёшь добро (или правишь) — пока это дёшево, до кода.

на фиче · одна задача в формате writing-plans
### Task 1: Сборка книги и роут экспорта
Files:
  Create  app/resplan/web/export_resplan.py
  Modify  app/resplan/web/app.py            # +роут /export/resplan.xlsx
  Test    tests/test_export_resplan.py
Interfaces · Produces: build_resplan_workbook(scope) → Workbook

- [ ] Шаг 1. Написать падающий тест
      assert wb.sheetnames == ["проекты","ресурсы","СВОД"]
- [ ] Шаг 2. Запустить, убедиться что падает
      Run: pytest tests/test_export_resplan.py -v
      Expected: FAIL (ImportError: build_resplan_workbook)
- [ ] Шаг 3. Минимальная реализация   # openpyxl, фильтр по роли
- [ ] Шаг 4. Запустить, тест зелёный
      Run: pytest -q  →  Expected: PASS
- [ ] Шаг 5. Коммит
4

Реализация

subagent-driven-developmentагент сам

Агент идёт по плану задача за задачей: сначала тест, потом код, сам себя ревьюит на каждом шаге и коммитит. Тебя между задачами не дёргает — остановится и спросит, только если упрётся в неясность.

на фиче · на каждом шаге
тест (RED) → код (GREEN) → ревью → коммит
5

Влить

finishing-a-development-branchнужен ты

Сначала прогоняет тесты (должны пройти), потом предлагает 4 варианта: merge / PR / keep / discard. Решаешь ты (для resplan main = прод — поэтому PR, а не merge).

на фиче · финал
✓ pytest: 1 passed
finishing → варианты: merge / PR / keep / discard
выбрано: PR (main = прод, не мёржим) · worktree сохранён для итераций по PR

Твоя роль — три точки

  • Обсудить — ответить на вопросы агента и согласовать спеку.
  • Ревью плана — прочитать план и дать добро (или поправить) до кода.
  • Финал — решить, как влить: merge / PR / keep / discard.

Всё между этими точками — ветку, тесты, код, ревью — агент делает сам, по флоу, не убегая в сторону.
